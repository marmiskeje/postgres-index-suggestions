Log format

Log line must have the following format: log_line_prefix = '^]%n¤%p¤%a¤%u¤%d¤%r¤%c¤%l¤%v¤%x¤'
Default separator is: ¤. Default indicator for log entry beginning is: ^]. These strings can be changed in Diploma thesis collector configuration. Also log file encoding.

STATISTICS

Relations, materialized views, indices, stored procedures

These statistics are generally collected only when The statistics collector is enabled:
track_activities = on
track_counts = on

Collecting of functions / stored procedures statistics needs further configuration:
track_functions = all # possibilities: none, pl, all

Be aware that PostgreSQL will never log inline functions, even when track_functions is enabled.

Statements

If statements are logged to file processed by Diploma thesis collector, statistics for statements will be generated.
Enabling of logging to file: logging_collector = on
Specification of logged statements: log_statement = all # possibilities: none, ddl, mod, all
Enabling of logging duration for duration stats: log_duration = on
Enabling of logging of execution plan for cost stats: debug_print_plan = on
Enabling of logging of rewritten query tree for object usage in statement stats: debug_print_rewritten = on

Be aware that when some debug_% option is enabled, pretty print must be disabled: debug_pretty_print = off
If you want to specify minimum duration for statement to be logged, use the following: log_min_duration_statement = number_in_milliseconds # -1 = disabled, 0 = log all statements

ANALYSIS

In order to be funcional, everything specified above must be enabled.
Specifically:

  • Analysis of unused objects needs statistics for relations, mat. views, indices, stored procedures and also logging basic logging of statements.
  • Analysis for indices or horizontal partitionings needs complete logging of statements (only logging of execution plans is optional).

For analyses purposes: HypoPG must be installed, at least in version 2.0.0.
